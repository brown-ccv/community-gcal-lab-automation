# Customization Examples

This document shows common customizations you might want to make.

## Changing follow-up intervals

Edit `src/calendar.js` and modify the `FOLLOW_UP_TYPES` array:

```javascript
// Current default (line ~8):
const FOLLOW_UP_TYPES = [
  { label: '1 day', days: 1 },
  { label: '10 day', days: 10 },
  { label: '45 day', days: 45 },
];

// Example: Add 7-day post follow-up
const FOLLOW_UP_TYPES = [
  { label: '1 day', days: 1 },
  { label: '10 day', days: 10 },
  { label: '45 day', days: 45 },
  { label: '7 day post', days: 52 },  // 45 + 7 days after base date
];

// Example: Different intervals
const FOLLOW_UP_TYPES = [
  { label: '1 week', days: 7 },
  { label: '2 week', days: 14 },
  { label: '1 month', days: 30 },
  { label: '3 month', days: 90 },
];
```

## Changing event duration

Edit `src/calendar.js`, line ~6:

```javascript
// Current default:
const DEFAULT_DURATION_MINUTES = 30;

// Change to 1 hour:
const DEFAULT_DURATION_MINUTES = 60;

// Change to 15 minutes:
const DEFAULT_DURATION_MINUTES = 15;
```

## Changing default time

Edit `src/cli.js`, line ~6:

```javascript
// Current default:
const DEFAULT_TIME = '09:00';

// Change to afternoon:
const DEFAULT_TIME = '14:00';
```

## Changing timezone

Edit `src/calendar.js`, line ~5:

```javascript
// Current default:
const TIMEZONE = 'America/New_York';

// Change to Pacific:
```

## Adding multiple attendees

Edit `src/cli.js` to modify the `DEFAULT_ATTENDEE` constant, or change `src/calendar.js` to accept an array:

### Option A: Single attendee (current)
```javascript
// In src/cli.js, line ~7:
const DEFAULT_ATTENDEE = 'gregory.lazatin2006@gmail.com';
```

### Option B: Multiple attendees
In `src/cli.js`, replace line ~7:
```javascript
const DEFAULT_ATTENDEES = [
  'gregory.lazatin2006@gmail.com',
  'colleague@example.com',
];
```

Then in `src/calendar.js`, around line ~65, change:
```javascript
// From:
attendees: [{ email: attendeeEmail }],

// To:
attendees: attendeeEmail.map(email => ({ email })),
```

And update the function call in `src/cli.js` (~line 132):
```javascript
// From:
attendeeEmail: DEFAULT_ATTENDEE,

// To:
attendeeEmail: DEFAULT_ATTENDEES,
```

## Adding email reminders

Edit `src/calendar.js`, around line ~78:

```javascript
// Current (no reminders):
reminders: {
  useDefault: false,
  overrides: [],
},

// Add 1 day before reminder:
reminders: {
  useDefault: false,
  overrides: [
    { method: 'email', minutes: 24 * 60 },  // 1 day before
  ],
},

// Add multiple reminders:
reminders: {
  useDefault: false,
  overrides: [
    { method: 'email', minutes: 24 * 60 },     // 1 day before
    { method: 'popup', minutes: 60 },          // 1 hour before
    { method: 'email', minutes: 10 * 24 * 60 }, // 10 days before
  ],
},
```

## Changing calendar target

By default, events are created in the "primary" calendar. To use a different calendar:

### Option 1: Use a specific calendar
Find your calendar ID in Google Calendar settings, then edit `src/cli.js`:

```javascript
// Around line ~132 and ~163, change:
calendarId: 'primary',

// To:
calendarId: 'your-calendar-id@group.calendar.google.com',
```

### Option 2: Make it configurable
Add a prompt in `src/cli.js` interactive mode to ask for calendar ID.

## Customizing event titles

Edit `src/calendar.js`, around line ~58:

```javascript
// Current format:
const eventTitle = `${title} - ${followUp.label} check-in`;
// Example: "BURST-001 - 10 day check-in"

// Alternative formats:
const eventTitle = `${title} Check-In (${followUp.label})`;
// Example: "BURST-001 Check-In (10 day)"

const eventTitle = `[${followUp.label.toUpperCase()}] ${title}`;
// Example: "[10 DAY] BURST-001"

const eventTitle = `${title} Follow-Up: ${followUp.label}`;
// Example: "BURST-001 Follow-Up: 10 day"
```

## Adding event descriptions/checklists

Edit `src/calendar.js`, around line ~73:

```javascript
// Current:
description: `Automated check-in event created for ${title}.\nBase date: ${baseDate}\nFollow-up type: ${followUp.label}`,

// Add checklist:
description: `Automated check-in event for ${title}

Base date: ${baseDate}
Follow-up type: ${followUp.label}

Checklist:
- [ ] Review participant status
- [ ] Check compliance
- [ ] Update records
- [ ] Send reminder if needed`,

// Add more structured info:
description: `## ${followUp.label} Check-In

**Participant:** ${title}
**Base Date:** ${baseDate}

### Tasks
1. Contact participant
2. Verify completion
3. Document results

*Auto-generated by Lab Calendar Automation*`,
```

## Disabling send invites (create events silently)

Edit `src/calendar.js`, around line ~101:

```javascript
// Current (sends invites):
sendUpdates: 'all',

// Don't send invites:
sendUpdates: 'none',

// Only notify external attendees:
sendUpdates: 'externalOnly',
```

## After making changes

After editing any `.js` files, simply re-run the script. No rebuild needed:

```bash
node src/cli.js
```
